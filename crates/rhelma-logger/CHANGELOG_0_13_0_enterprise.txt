Rhelma-LOGGER CHANGELOG

[0.13.0-enterprise] - 2025-12-02
-------------------------------

Added
-----

- LoggerConfig::from_unified(UnifiedObservabilityConfig, Option<CoreConfig>)
  for thin integration with rhelma-config v3.4.0-enterprise.
- RhelmaLogger::init_from_unified(...) convenience entrypoint, wiring logger
  initialisation directly from rhelma-config models.
- Public re-export of FileServiceDispatcher and dispatcher modules so that
  dispatchers are aligned with the unified observability stack.

Changed
-------

- Logger configuration is now driven primarily from UnifiedObservabilityConfig
  and CoreConfig, rather than ad-hoc environment parsing.
- Queue sizing and backpressure options are derived from PerformanceProfile:
  LowLatency  -> Sync, small queue, DropNewest
  Balanced    -> Async, medium queue, DropNewest
  HighThroughput -> Async, large queue, DropOldest

Notes
-----

- Existing APIs (LoggerConfig fields, builder macros, dispatchers) remain
  backwards compatible at the conceptual level.
- Tests around basic initialisation and builder behaviour are provided.
